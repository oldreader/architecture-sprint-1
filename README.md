Текущее решение - монолит
- 
Для декомпозиции будем использовать DDD. Выдели основные домены:
1) Домен управления пользователем
2) Домен выдачи фида фотографий. Фид может независимо генерироваться в зависимости от решений, которые требуются бизнесу. 
3) Домен загрузки фотографий. Загрузка контента выглядит, как кандидат на отдельный домен.  
4) Домен управления лайками. Это отдельный домен, который может быть оптимизирован на уровне отдельной команды разработки. 

Целевое видение микрофронтендов 
-
Выделить 4 микрофронтенда по: 
1) [home](frontend/microfrontend/home) - микрофронтенд для сборки приложения
2) [card-board](frontend/microfrontend/card-board) - микрофронтенд выдачи фида
3) [photo-manager](frontend/microfrontend/photo-manager) - микрофронтенд для загрузки фотографий
4) [user](frontend/microfrontend/user) - микрофронтенд для управления пользователями
5) [likes](frontend/microfrontend/likes) - микрофронтенд для управления лайками

Выбор фреймворка для решения
-
Всё приложение написано на react, поэтому будет проще и эффективнее решать построение архитектуры на webpack federation framework. 
Он позволит нам управлять разрешением конфликтов между библиотеками на уровне сборки и запуска приложений. 
Для микрофронтендов будем использовать create-mf-app шаблон для приложений. 
Так как он генерирует шаблон для TS, то импортировать компоненты из текущей реализации будем в JS, выключив ограничения "allowJs": true в [tsconfig.json](frontend/microfrontend/home/tsconfig.json).
Это позволит нам не переписывать реализацию на TS. 

План миграции
-
Миграция будет происходить в рамках текущего проекта в модуле [frontend](frontend) в каталоге [microfrontend](frontend/microfrontend). 
Будут выделены микрофронтенды согласно Доменной модели. 
1) В микрофронтенд для управления пользователями переезжают компоненты: Footer, Header, Login и Register. Так же сюда должна переехать часть по редактированию пользователя из Main
2) Микрофронтенд выдачи фида: сюда еперезжает часть с карточками из Main и сами карточки Card
3) Микрофронтенд для загрузки карточек AddPlacePopup 
4) Микрофронтенд для лайков - необходимо отделить лайки от карточки, так как это разные домены.
Приложение фида фотографий [card-board](frontend/microfrontend/card-board) будет зависеть от лайков [likes](frontend/microfrontend/likes) 

**3й шаг не готов. Приложение не готово к запуску.**
